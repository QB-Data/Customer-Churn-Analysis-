# Customer Churn Analysis

Predicting which telecom customers are likely to cancel their service — so the business can step in early and keep them.

## What Is Customer Churn?

Customer churn is when people stop using a company's product. For a telecom company, that means a customer cancels their phone or internet plan. Even a small reduction in churn can translate to significant revenue savings, which is why companies invest in understanding *why* customers leave and *who* is most likely to leave next.

This project uses machine learning to answer both of those questions.

## The Dataset

The [Telco Customer Churn](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) dataset from Kaggle contains **7,043 customer records** with 21 attributes including:

| Category | Examples |
|---|---|
| Demographics | Gender, senior citizen status, partner, dependents |
| Services | Phone, internet type, streaming, tech support, online security |
| Account info | Tenure (months as customer), contract type, billing method, monthly/total charges |
| Target | **Churn** — did the customer leave? (Yes / No) |

## Approach

1. **Data cleaning** — fix missing values, convert types, drop IDs
2. **Exploratory analysis** — visualize patterns in who stays vs. who leaves
3. **Feature encoding** — convert text categories into numbers the model can process
4. **Decision Tree model** — a classifier that learns a series of yes/no rules from the data (think of it like a flowchart that asks questions about a customer and arrives at a prediction)
5. **Evaluation** — measure accuracy, precision, recall, and F1 score on held-out test data

### Why a Decision Tree?

Decision Trees are **interpretable** — you can look at the tree and understand exactly *why* the model made a specific prediction. This matters in a business setting where stakeholders need to trust and act on the results, not just see a number.

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/QB-Data/Customer-Churn-Analysis-.git
cd Customer-Churn-Analysis-
pip install -r requirements.txt
```

### 2. Download the dataset

```bash
python src/download_data.py
```

> Requires a [Kaggle account](https://www.kaggle.com/). The `kagglehub` library handles authentication — follow its prompts on first run.

### 3. Run the analysis

**Option A — Notebook** (recommended for interviews and presentations):
```bash
jupyter notebook churn_analysis.ipynb
```

**Option B — Terminal** (quick run, prints results and saves charts):
```bash
python run_pipeline.py
```

## Key Findings

| Finding | What It Means |
|---|---|
| **Contract type** is the top predictor | Month-to-month customers churn at much higher rates — locking in longer contracts reduces risk |
| **New customers churn the most** | The first few months are critical — strong onboarding and early engagement matter |
| **Higher monthly charges correlate with churn** | Customers paying more may feel they're not getting enough value — worth proactive outreach |
| **Fibre optic users churn more** | Could indicate service quality or pricing issues specific to that tier |
